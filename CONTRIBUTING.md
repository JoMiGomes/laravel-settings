# Contributing to Laravel Settings

Thank you for considering contributing to the Laravel Settings package!

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue on Bitbucket with:
- A clear description of the problem
- Steps to reproduce the issue
- Expected vs actual behavior
- Your environment details (PHP version, Laravel version, etc.)

### Suggesting Features

Feature suggestions are welcome! Please open an issue describing:
- The feature you'd like to see
- Why it would be useful
- How it might work

### Pull Requests

1. Fork the repository
2. Create a new branch for your feature or bugfix
3. Write tests for your changes
4. Ensure all tests pass
5. Follow the existing code style
6. Submit a pull request with a clear description of your changes

### Development Setup

```bash
# Clone the repository
git clone https://bitbucket.org/jomigomes/laravel-settings.git

# Install dependencies
composer install

# Run tests
composer test
```

### Coding Standards

- Follow PSR-12 coding standards
- Write clear, descriptive commit messages
- Add tests for new features
- Update documentation as needed

### Testing

All contributions should include tests. Run the test suite with:

```bash
composer test
```

## Code of Conduct

Please be respectful and constructive in all interactions.

Thank you for contributing!
